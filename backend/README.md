# Backend Demo for Jury

## What this backend does
- A tiny **Express** server that uses **SQLite** (`better-sqlite3`) to store a simple `jobs` table.
- Provides a **read‑only** endpoint `GET /jobs` that returns all rows as JSON.
- Includes a **seed** endpoint `POST /seed` (optional) to add demo rows while you are preparing the demo.
- Runs on **port 4000** so it does not clash with the Vite dev server (5173).

## How to start it
1. Ensure you have the project dependencies installed:
   ```bash
   npm install
   ```
2. Install the extra backend dependencies (Express, CORS, better‑sqlite3):
   ```bash
   npm install express cors better-sqlite3
   ```
3. Start the backend with the script we added to `package.json`:
   ```bash
   npm run backend
   ```
   You should see a line like:
   `🚀 Backend API listening on http://localhost:4000`

## Viewing the saved data
- Open your browser and navigate to **http://localhost:4000/jobs**.
- You will receive a JSON array containing the rows from the `jobs` table.
- If the table is empty, you can add a demo record via the seed endpoint:
   ```bash
   curl -X POST http://localhost:4000/seed \
        -H "Content-Type: application/json" \
        -d '{"title":"Full‑Stack Engineer","company":"Acme Corp","location":"Remote"}'
   ```
   Then refresh the `/jobs` page to see the new entry.

## How this fits into the existing project
- The backend lives in a **new `backend/` folder** and does **not modify any frontend code**.
- It is completely independent of the Vite/React stack, so the UI you have built continues to work unchanged.
- The jury can run the backend alongside the React dev server to demonstrate that data is persisted and can be inspected.
